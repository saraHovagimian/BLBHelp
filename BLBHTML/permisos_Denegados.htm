Permisos Denegados

Este informe permite visualizar la configuración de los permisos denegados de los usuarios y de los grupos.

Para acceder a este punto el usuario debe pertenecer al grupo Administradores.

 Usuario o grupo

El informe puede emitirse por usuario o por grupo de usuarios. Para ello se debe indicar el usuario o el grupo deseado en los campos correspondientes.

 Usuario

Si se desea generar el informe para un usuario se debe activar este campo y seleccionar un usuario.

 Grupo

Si se desea generar el informe para un grupo se debe activar este campo y seleccionar un grupo.

 Permisos por

Se debe indicar mediante tildes cuales son los elementos del sistema (Atributo, Centro, Legajo, Concepto, Variable y Suceso) sobre los que se desean ver los permisos denegados.

 Atributo / Centro / Legajo / Concepto / Variable / Suceso

Si el campo esta tildado, se emite el informe contemplando la definición de permisos para el elemento seleccionado.

 Destino

 Para tener en cuenta

Para acceder a este punto el usuario debe pertenecer al grupo Administradores.

Este informe sólo exhibe los permisos denegados, es decir, si el usuario o grupo seleccionado no tiene tildes en ningún casillero de la columna Denegado, el informe no se emite y se muestra un mensaje advirtiendo tal situación.

El informe se emite por un usuario o por un grupo, no se pueden seleccionar mas de un usuario o mas de un grupo, ni una combinación de ambos.
