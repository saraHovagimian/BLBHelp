Datos Históricos

 Usar Datos Históricos

Al activar este campo, se está indicando que se requiere emitir un informe con datos históricos. En ese caso, puede utilizarse una reconstrucción ya existente en el sistema, o crear una nueva.

Si no está activado, el informe exhibe información sobre datos actuales de la base de datos.

 Históricos Reconstruidos

En este cuadro es posible especificar las referencias a grupos de datos para emitir el informe, así como también, crear nuevas referencias de datos o eliminar existentes.

 Disponibles

En este campo figuran las referencias asociadas a un conjunto de datos específico, estos conjuntos de datos requeridos para efectuar las reconstrucciones son manejados internamente por BAS Laboro, agrupándolos en grupos llamados Sec1, Sec2, etc.

 Eliminar

Permite eliminar grupos de datos almacenados para la reconstrucción de datos. Cuando se realiza una reconstrucción, se crean las tablas con la misma estructura que las originales y con la información histórica. Si esta reconstrucción involucra muchos datos tendrá un tamaño considerable y, si se piensa que es posible tener tantas reconstrucciones como se quiera, la cantidad de datos puede crecer mucho impactando sobre la performance del sistema.

Por lo cual, cuando a una reconstrucción no se le dé más uso, se recomienda eliminarla.

Para ello, en el campo"Disponibles" de alguno de los informes que utilizan la reconstrucción a eliminar, o en la tarea "Administración de auditorias", se debe seleccionar la reconstrucción que corresponda.

Una vez seleccionada, bastará presionar el botón "Eliminar" y, previo pedido de confirmación, BAS Laboro eliminará la reconstrucción seleccionada

 Crear Nuevo

Permite crear una nueva reconstrucción de datos. Al hacerlo se abre la ventana "Reconstruir Históricos " que permite informar la fecha a la cual se desea reconstruir los datos requeridos para el informe.

BAS Laboro asigna, en forma automática, una descripción a cada una de las reconstrucciones que se realizan. El nombre que se asigna está compuesto por:

  • Código de reconstrucción

Este código identifica al grupo de informes que comparten el mismo conjunto de datos reconstruidos. No reviste interés para el usuario, excepto a título informativo para saber qué informes comparten las mismas reconstrucciones.

  • Nombre del informe y fecha a la cual se generó

Si no se ingresa ningún texto en el campo "Descripción", se exhibe "Sin descripción".

Al solicitar la reconstrucción es posible indicar algunos filtros de modo de reconstruir, solamente, la información de algunos legajos.

Los filtros se establecen en la ficha "Legajos" del informe del cual se trate, indicando los códigos de legajos cuya información se desea reconstruir. Vale aclarar que, al tratarse de procesos de reconstrucción, no está prevista la posibilidad de filtrar la información por atributos de legajos; solamente, se pueden establecer filtros en base a sus códigos.

Esta limitación puede entenderse si se piensa que los valores de atributos que se asignan a los legajos pueden sufrir modificaciones a lo largo del tiempo, por lo cual, si se indicasen filtros en base a dichos valores, el resultado que se obtendría no sería confiable.

La posibilidad de utilizar filtros en relación a los legajos es sumamente útil ya que un proceso de reconstrucción que abarque un período de tiempo prolongado, puede llevar un tiempo considerable. Por lo cual, en caso que no se requiera reconstruir todos los datos, se recomienda utilizar los filtros para limitar la información a reconstruir, solamente a la requerida.

 Para tener en cuenta

Para utilizar los datos almacenados y poder reconstruir la información de un listado a determinada fecha, se deben reconstruir las tablas de la base de datos a esa fecha.

El proceso de reconstrucción, en base a la fecha a la que se solicita el informe, construirá réplicas de las tablas que contienen los datos que se requieren para emitir el listado e cuestión.

Es importante recalcar que los datos existentes en las tablas al momento de la reconstrucción no son reemplazados, sino que las reconstrucciones se generan en otras tablas que no afectan, en absoluto, a las tablas originales.

Cuando un usuario realiza una reconstrucción, ésta quedará disponible para todos los usuarios y para todos los listados que utilicen ese mismo conjunto de datos.

Todos los informes que prevén la reconstrucción histórica de datos trabajan en forma similar a la que veremos, a continuación, para este informe.

Si se requiere retrotraer la información a alguna fecha pasada, es posible efectuar una reconstrucción de los datos a esa fecha, independientemente de los valores existentes a la fecha en que se solicita dicha información.
