Definir seguridad

La asignación de permisos a los usuarios (en forma individual o por pertenecer a determinados grupos) permite limitar los legajos que podrán consultar, modificar o liquidar y también las novedades que podrán informar y consultar.

El único usuario autorizado para definir los permisos es sa.

Mediante la asignación de permisos es posible establecer a qué usuarios (o grupos de usuarios) se les permite, o no, realizar determinadas acciones (ver, modificar, liquidar, etc.) sobre determinados elementos del sistema (legajos, variables, sucesos, etc.).

Los elementos del sistema sobre los que se pueden establecer permisos son los siguientes:

Valores de atributos (sólo para los atributos definidos como "Habilitados para seguridad")

Centros de apropiación

Legajos

Conceptos

Variables

Sucesos

Para cada uno de estos elementos se pueden asignar o denegar permisos sobre un grupo de acciones según se muestra en los cuadros siguientes.

La aplicación dispone de un usuario, llamado sa, que cuenta con todos los permisos, sin excepción. Este usuario no puede eliminarse y es el autorizado, inicialmente, a asignar o denegar permisos al resto de usuarios y grupos del sistema.

También existe un grupo llamado Administradores, este grupo cuenta con todos las acciones permitidas, sin excepción. Los usuarios que pertenecen a este grupo son los únicos autorizados a asignar o denegar permisos al resto de los usuarios. El grupo puede tener tantos usuarios como se desee, inicialmente el único usuario incluido en este grupo es sa.

La forma de asignar un permiso consiste en indicar para un usuario, o para un grupo de usuarios, si se permite o se deniega realizar las acciones que correspondan.

Existe un grupo Todos al que, obligatoriamente, pertenecen todos los usuarios (excepto el usuario sa). Este grupo se actualiza automáticamente con cada alta de usuario y no se podrá excluir ningún usuario de él. Inicialmente, tiene todos los permisos habilitados.

Si no se han configurado permisos específicos para ningún usuario, o grupo de usuarios, todos los usuarios toman, por defecto, los permisos de seguridad asignados al grupo Todos que, como mencionamos, inicialmente tiene todo permitido, al igual que el usuario sa.

Luego, a medida que se vayan asignando permisos se irán modificando los permisos de cada usuario en base a los que se le asignen a él específicamente o a los que le correspondan por pertenecer a determinados grupos. A los usuarios que pertenecen al grupo Administradores no se les puede modificar la definición de permisos. Estos tienen, mientras pertenezcan a Administradores, todas las acciones permitidas, sin excepción.

La definición de permisos por usuarios prevalece sobre la definición de permisos por grupos. Por ejemplo, supongamos que un grupo tiene permisos denegados para Informar novedades del suceso Vacaciones; por lo tanto, en principio, ningún usuario que se encuentre dentro de este grupo podrá informar novedades de vacaciones. Supongamos también que existe un usuario, dentro del mismo grupo, que sí tiene permiso para Informar novedades, entonces el usuario podrá informar las novedades correspondientes al suceso Vacaciones ya que sus permisos prevalecen sobre los permisos del grupo.

En caso que a un usuario no se le hayan asignado permisos en forma individual y que pertenezca a varios grupos teniendo permisos diferentes, prevalecen los permisos denegados.

Los permisos pueden definirse desde la tabla Usuarios y desde la tabla Grupos. En estos casos se selecciona un usuario o un grupo de usuarios y se indican las acciones permitidas y denegadas para los distintos elementos del sistema.

También es posible configurar los permisos desde las tablas Legajos y Centros de apropiación. En estos casos se indica un legajo o un centro de apropiación y se definen qué usuarios o grupos de usuarios tienen acciones permitidas o denegadas.

Al momento de definir los permisos se debe tener en cuenta que los usuarios que pertenecen al grupo Administradores tienen, mientras están en el grupo, todas las acciones permitidas, sin excepción.

Independientemente de las tablas desde las cuáles se asignen los permisos (Usuarios - Grupos - Legajos o Centros de apropiación), el resultado final de las acciones permitidas y denegadas es el mismo y podrá visualizarse ingresando por cualquiera de estos puntos del menú. El asignar permisos desde unas u otras tablas dependerá de las preferencias de operación del usuario.

Cómo definir los permisos

Para un ítem:

  - Ir a la solapa correspondiente.

  - Seleccionar el ítem al que se le desea definir los permisos.

  - Tildar o destildar los casilleros de las columnas Permitir o Denegar, según corresponda.

  - Presionar el botón OK.

Para más de un ítem:

  - Ir a la solapa correspondiente.

  - Seleccionar los ítems a los que se les desean definir los permisos, utilizando las teclas Control o Shift.

  - Presionar el botón Actualizar que se despliega en la grilla inferior

  - Tildar o destildar los casilleros de las columnas Permitir o Denegar, según corresponda.

  - Presionar el botón OK.

La primera acción de la grilla se llama Todas y permite quitar o colocar un tilde en todas las filas de la columna, por ejemplo, asignar a todas las acciones Permitir.

Se debe tener en cuenta que Todas sólo puede usarse si no hay ningún tilde gris en la grilla. Si en la fila hay algún tilde gris no se puede cambiar el permiso sin antes sacarlo.

Vale aclarar que al quitar el estado gris, no se puede volver a obtenerlo. Para sacar el tilde gris se debe verificar, para cada uno de los ítems seleccionados, cual es el estado de permisos que posee cada uno de ellos y tildar o destildar los casilleros para que todos tengan la misma configuración (tildados o destildados). De esta manera, al seleccionar el conjunto de ítems, en caso de quedar todos los ítems tildados el tilde no aparece gris y en caso de quedar todos los ítems destildados el casillero aparece en blanco; por lo tanto, en ambos casos, el estado puede modificarse libremente.

Usuarios y grupos

  Definir seguridad

Tanto en la tabla Usuarios como en la tabla Grupos, se incluye el botón Definir seguridad. Este botón permite acceder a la ventana de configuración de permisos.

Los únicos usuarios autorizados a configurar los permisos son los que pertenecen al grupo Administradores.

Para que se active el botón Definir seguridad, primero se debe indicar el usuario o grupo para el que se desean configurar los permisos. Luego, presionándolo, se accede a la ventana Permisos.

 Usuario o grupo

En la parte superior izquierda se visualiza una imagen y el nombre del usuario o del grupo. La imagen varía ya sea que se trate de un usuario o de un grupo.

Se pueden configurar permisos para cada uno de los ítems de cada grilla superior, en forma individual, o para más de un ítem, seleccionándolos mediante las teclas Control o Shift.

 Atributos (sólo los definidos como Habilitados para seguridad)

Hay una solapa por cada atributo definido como habilitado para seguridad. Estas solapas están identificadas con el nombre del atributo.

 Valores de atributos

En la grilla superior se exhiben todos los valores que puede tomar el atributo.

 Centros de apropiación

Cada centro de apropiación tiene su propia solapa y están identificadas con la denominación de cada uno de ellos.

 Centros de apropiación

En la grilla superior de cada solapa se exhiben los distintos centros creados.

La configuración que se defina para el usuario o grupo desde esta solapa también se exhibe (para el usuario o el grupo) si se accede a la definición de permisos, mediante el botón Definir seguridad, desde la tabla Centros de apropiación

 Lista de legajos

Hay una única solapa para los legajos con el nombre Lista de legajos.

 Legajos

La grilla superior inicialmente está vacía. Para incluir uno o más legajos se debe presionar el botón Agregar, seleccionar los legajos a los que se les van a configurar permisos y presionar el botón Aceptar. Si hay legajos incluidos en la grilla superior, es posible borrarlos mediante los botones Borrar y Borrar todos. Los legajos borrados pierden todos los permisos configurados.

 Conceptos

En esta solapa existe la posibilidad de seleccionar que Tipos de conceptos se desean ver; para ello se incluye un combo con los tipos de conceptos existentes (haberes, retenciones, salario familiar, redondeo y contribuciones).

 Conceptos

En la grilla se exhiben los conceptos de acuerdo al tipo seleccionado. Inicialmente se exhiben todos los conceptos.

 Variables

Hay una única solapa para las variables.

 Variables

Se exhiben todas las variables existentes en el sistema, tanto las predefinidas como las definidas por el usuario

 Sucesos

Hay una única solapa para los sucesos. En esta solapa existe la posibilidad de seleccionar que Tratamientos se desean ver; para ello se incluye un combo con los tratamientos existentes.

 Sucesos

En la grilla se exhiben los sucesos de acuerdo al tratamiento seleccionado. Inicialmente se exhiben todos los sucesos.

 Acciones

En esta grilla se exhiben los permisos que posee el ítem, o ítems, seleccionados. Posee tres columnas; debajo del título Acciones se ven las tareas sobre las que se pueden establecer permisos, éstas varían de acuerdo al elemento del sistema (solapa) en que se encuentre posicionado. En las columnas rotuladas Permitir y Denegar se establecen los permisos para cada una de las tareas mediante tildes.

Cada casillero puede tener tres estados:

 Sin tilde: Indica que el ítem no posee configuración para esa acción en particular.

 Con tilde: Indica que el ítem posee configuración para esa acción en particular.

 Con tilde gris (sólo cuando se selecciona más de un ítem): indica que, dentro de los ítems seleccionados, alguno tiene ese casillero con tilde y otros sin tilde.

Se debe tener en cuenta que para un mismo ítem una acción no puede tener ambos casilleros con tilde.

Legajos y Centros de apropiación

  Definir seguridad

Tanto en la tabla Legajos como en la tabla Centros de apropiación, se incluye el botón Definir seguridad. Este botón permite acceder a la ventana de configuración de permisos. Este botón se habilita luego de indicar un legajo y, únicamente, cuando el usuario pertenece al grupo Administradores.

Para que se active, primero se debe indicar el legajo, o centro de apropiación, para el que se desean configurar los permisos, luego, presionándolo, se accede a la ventana Seguridad.

 Nombre

En la parte superior izquierda se visualiza una imagen y el nombre del legajo o el del centro de apropiación. La imagen varía ya sea que se trate de un legajo o de un centro de apropiación.

Para asignar los permisos puede seleccionarse un usuario, un grupo o varios de ellos mediante las teclas Control o Shift. Al seleccionar más de un usuario o grupo, se debe presionar el botón Actualizar, que se despliega en la grilla inferior, para refrescar los permisos de los mismos. El grupo Todos no puede seleccionarse junto con otros grupos o usuarios.

Para agregar usuarios o grupos, se debe presionar le botón Agregar, seleccionar un usuario, un grupo o varios de ellos, mediante las teclas Control o Shift y presionar el botón Aceptar.

Para borrar uno o más usuarios o grupos, se deben seleccionar los mismos y presionar el botón Borrar. Mediante el botón Borrar todos es posible borrar todos los usuarios y grupos exhibidos.

El grupo Todos siempre aparece en esta grilla y no puede borrarse.

 Acciones

En esta grilla se exhiben los permisos que posee el usuario o el grupo seleccionado. Posee tres columnas; debajo del título Acciones se ven las tareas sobre las que se pueden establecer permisos. En las columnas rotuladas Permitir y Denegar se establecen los permisos para cada una de las tareas mediante tildes.

Cada casillero puede tener tres estados:

 Sin tilde: Indica que el usuario (o grupo) no posee configuración para esa acción en particular.

 Con tilde: Indica que el usuario (o grupo) posee configuración para esa acción en particular.

 Con tilde gris (sólo cuando se selecciona más de un usuario o grupo): indica que, dentro de los usuarios o grupos seleccionados, alguno tiene ese casillero con tilde y otros sin tilde.

Se debe tener en cuenta que, para un mismo usuario o grupo, una acción no puede tener ambos casilleros con tilde.

 Para tener en cuenta:

Si se desean definir permisos para un usuario incluido en el grupo Administradores, al presionar el botón Definir seguridad aparece una mensaje haciendo saber que a ese usuario no se le pueden definir permisos porque está incluido en Administradores. Esto mismo ocurre si, desde la tabla Grupos, se presiona este botón para el grupo Administradores. Esto se debe a que este grupo y los usuarios que incluye, siempre, sin excepción, tienen todas las acciones permitidas.

Es importante tener en cuenta a que empresas pertenecen el usuario y el grupo, ya que la definición de permisos se realiza por usuario (o por grupo) por cada empresa. Es decir, si un usuario (o grupo) pertenece a más de una empresa y se desea definirle permisos en todas las empresas, la definición de permisos se debe realizar tantas veces como sea necesario, ingresando, en cada caso, a la empresa correspondiente.

Independientemente de las tablas desde las cuáles se asignen los permisos (Usuarios - Grupos - Legajos o Centros de apropiación), el resultado final de las acciones permitidas y denegadas es el mismo y podrá visualizarse ingresando por cualquiera de estos puntos del menú. El asignar permisos desde unas u otras tablas dependerá de las preferencias de operación del usuario.
