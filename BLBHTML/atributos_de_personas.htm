Atributos de personas

Los atributos permiten asignar a las personas diferentes características destinadas a agruparlas o a filtrarlas en diversas tareas del sistema.

Definir un atributo consiste en definir una característica distintiva que interese aplicar a las personas y los valores que ésta puede asumir para cada uno de ellas.

Los atributos, al igual que las personas, son compartidos por todas las empresas habilitadas en la base de datos.

Los atributos que se habilitan son de uso es obligatorio para todas las personas que se definan.

Con esto queremos significar que al incorporar una persona a la base de datos se le debe asignar un valor determinado para cada uno de los atributos habilitados.

Es por ello que es necesario indicar, en relación a cada atributo, un valor del mismo como valor Por defecto. Dicho valor es el que el sistema asignará, en forma automática, a las nuevas personas que se incorporen.

 Código

Hasta tres caracteres que identifican al atributo.

Por ejemplo NES.

 Descripción

Puede tener hasta 20 caracteres.

En relación al código NES mencionado, la descripción puede ser Nivel de estudios.

 Valores

Se deben registrar, uno a uno, los valores que puede asumir el atributo.

 Valores

 Código

Puede incluir hasta 3 caracteres.

Por ejemplo, en el caso del atributo NES-Nivel de estudios puede utilizarse, como codificación, S, T, TI, U, UI, etc.

 Descripción

Puede incluir hasta 25 caracteres.

En el caso de la codificación mencionada las descripciones pueden, Secundario para el código S, Terciario para el código T, Terciario incompleto para TI, Universitario para U y Universitario incompleto para UI.

 Por defecto

Se debe activar este campo en relación al valor del atributo que se desea asignar automáticamente a las personas que se incorporan al sistema.

 Comentarios

Los atributos que se habilitan son de uso es obligatorio para todas las personas que se definan.

Con esto queremos significar que cada persona que se incorpora al sistema debe tener asignado un valor determinado para cada uno de ellos.

Es por ello que al habilitar una persona se le asigna, para cada uno de los atributos definidos, el valor que se haya indicado para cada uno como valor Por defecto. Los valores asignados por defecto luego pueden ser reemplazados por los valores que correspondan a la persona del cual se trate.

Por lo cual, para cada atributo que se defina, habrá que analizar que valor es el más conveniente informar como valor Por defecto. En los casos en que al dar de alta a las personas no se conozca aun la posición de las mismas frente al atributo que se define, o se desee posponer el ingreso de dicha información para una instancia posterior, será conveniente incorporar un valor del mismo del tipo A determinar.

En cambio, si se encuentra que determinado valor de un atributo ocurre con mayor frecuencia que los restantes será conveniente (ahorrará trabajo) indicar dicho valor como valor Por defecto de modo que se asigne automáticamente. Luego, bastará con cambiar el valor del atributo en los legajos para los que el valor por defecto no aplique.

 Para tener en cuenta:

Para grabar un atributo en la tabla se requiere que se haya indicado un valor del mismo como valor Por defecto.
