Tipos de Liquidación

Los Tipos de liquidación que se habilitan en esta tabla se asociarán a cada liquidación que se realice fijando una serie de características y particularidades.

Al abrir una liquidación se debe informar el Tipo de liquidación al cual corresponde.

 Código

Código identificatorio del tipo de liquidación.

Por ejemplo, 1AQ, 2AQ, AGU, MEN, VAC, etc.

 Descripción

Texto que, conjuntamente con el código, identifica el Tipo de liquidación que se habilita.

En relación a los ejemplos de códigos citados, pueden corresponder como descripciones: Primera quincena, Segunda quincena, Aguinaldo, Liquidación mensual, Inf. de vacaciones.

 Liquidación final complementaria

Indica que en la liquidaciones de este tipo sólo podrán liquidarse legajos a los cuáles ya se les efectuó una liquidación final.

Cuando se realiza una liquidación final, se toma nota de los legajos a los cuáles se les efectúa. Luego, para dichos legajos, sólo se le podrá efectuar liquidaciones cuyos tipos se hayan definido con este activado.

Cuando se selecciona este campo, se deshabilitan automáticamente las opciones Liquidación SAC y Liquidación final.

 Liquidación S.A.C.

Si se activa esta opción se indica que en las liquidaciones a las cuáles se asocie este tipo se liquidará, entre otros conceptos, el SAC.

Al realizar una liquidación de SAC, el sistema toma nota del mes en el cual ésta se produce de modo de mantener actualizado el dato correspondiente a la cantidad de meses trabajados desde la última liquidación de SAC. Esto hace posible automatizar los cálculos cuando deba liquidarse el SAC nuevamente.

 Liquidación Final

Si se activa esta opción, las liquidaciones a las cuáles se asocie este tipo, serán consideradas por el sistema como liquidaciones finales.

Cuando se realiza una liquidación final se toma nota de los legajos a los cuáles se les efectúa. Luego, para dichos legajos, sólo se podrán efectuar liquidaciones finales complementarias, no siendo posible efectuarles ningún otro tipo de liquidación.

Nótese que el tener informada una fecha de egreso no inhabilita la liquidación de un legajo. La inhabilitación de liquidaciones, de tipo normal, viene dada por el hecho de haber efectuado al legajo una liquidación final.

 Liquidar únicamente legajos con novedades para este tipo de liquidación

Si se activa este campo, al realizar liquidaciones a las que se asocie este tipo, sólo se incluirán los legajos que tengan novedades informadas, específicamente, para este tipo de liquidación.

Esto es útil, por ejemplo, cuando en la implementación coexisten novedades habilitadas para todas las liquidaciones y novedades habilitadas para liquidaciones específicas, como por ejemplo, en el caso de las novedades relativas a vacaciones.

Efectivamente, al definir el tipo de liquidación a asociar a las liquidaciones de vacaciones, puede ser conveniente activar el campo Liquidar únicamente legajos con novedades para este tipo de liquidación. De este modo, si se informan por ejemplo vacaciones para 3 legajos en una nómina de 1000, en la liquidación de vacaciones, el sistema incluirá solamente las novedades relativas a esos 3 legajos, obviando todas las novedades que se encuentren vigentes y que sean válidas en todo tipo de liquidación.

 Liquidar únicamente legajos con novedades para este tipo de liquidación

Si se activa este campo, al realizar liquidaciones a las que se asocie este tipo, sólo se incluirán los legajos que tengan novedades informadas, específicamente, para este tipo de liquidación.

 Se deposita en cuenta bancaria alternativa

Si está activado, cuando se realice una liquidación de este tipo, para cada legajo que se liquide, el recibo incluirá, de las reparticiones bancarias que se le hayan asociado, aquella que corresponda a una cuenta bancaria alternativa.

 Actualiza Visor de 4ta categoría

Si se desea guardar la información de 4ta. categoría, este check box debe estar activado.

 Conceptos de liquidación a liquidar

Esta tabla exhibe los conceptos habilitados para liquidaciones que correspondan al tipo en cuestión.

La tabla es de doble entrada. Con esto queremos significar que la habilitación de un concepto para un tipo de liquidación puede informarse en esta tabla o en la ficha Tipos de liquidación de la tabla Conceptos de liquidación, en relación al concepto del que se trate.

Los conceptos que se informen en esta tabla se exhiben en la ficha Tipos de liquidación de la tabla Conceptos de liquidación y, recíprocamente, al habilitar un concepto para un tipo de liquidación, dicho concepto se exhibe en esta tabla en relación al tipo de liquidación para el cual se habilitó.

 Comentarios

Es posible realizar, dentro de un mes, todas las liquidaciones que se requieran. Para ello la única consideración a tener en cuenta es que deben habilitarse previamente en la tarea Cambio de liquidación indicando, entre otros datos, el Tipo de liquidación al cual corresponde.

Si bien no hay límite en la cantidad de liquidaciones que se pueden efectuar por mes, no es posible realizar dos liquidaciones del mismo tipo. Por lo cual si fuese necesario, por ejemplo, liquidar dos veces gratificaciones en un mismo mes, se deberán habilitar dos tipos de liquidación, por ejemplo, de códigos GR1 y GR2.

 Para tener en cuenta:

Cuando se incorpora un nuevo concepto de liquidación a la tabla Conceptos de liquidación, automáticamente, el concepto queda habilitado para ser liquidado en todos los tipos de liquidación existentes en el sistema al momento de su incorporación.

Es por este motivo que, al definir un concepto, la ficha Tipos de liquidación exhibe todas las liquidaciones habilitadas en el sistema con el campo Incluir activado. Si se desea que el concepto no se liquide en determinados tipos de liquidación bastará con desactivar dicho campo para los tipos de liquidación que correspondan.

En cambio, al definir un nuevo tipo de liquidación, los conceptos existentes en el sistema no se ven afectados, es decir, no quedan habilitados para ser liquidados en el nuevo tipo de liquidación definido.
